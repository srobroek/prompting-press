## ✅ What Went Well (keep doing)

1. **Analyze gate paid for itself.** F1 (provenance lint was *unimplementable* against the kernel's real
   surface — no "guard position" concept) and F4 (token hook was a bare seam adding little) were both
   caught at `/speckit.analyze` and resolved by `refine` BEFORE any code. Zero mid-build rework resulted.
2. **Constitution-driven design held cleanly.** The consumer delegates 100% to the kernel (Principle I) —
   sync.conflicts confirmed no render/agreement/hash logic was duplicated, and the FFI gate stayed green
   (Principle II). The narrow boundary (no I/O, no tokens) made the crate small and the audit fast.
3. **SEC-004 scrub was specified, implemented, and test-pinned** — three tests prove a secret in a kernel
   error never reaches a message or `Display`. Security-by-construction, not bolted on.
4. **Reviewing the diff, not the summary, caught real gaps.** CR-1 (check() double-analyzed a
   `default`-named variant and never analyzed its declared arm) and ER-1 (a `.expect()` panic path in
   `src`) were found by reading the actual code post-execution, then fixed and re-tested.
5. **Independent verification of every subagent claim** — the discipline that saved this cycle (below).

## ⚠️ What Could Improve (start/stop)

1. **Subagent tool-channel corruption is now systemic, not incidental.** This session BOTH sync agents
   (analyze + conflicts) returned output corrupted by a bash-channel glitch; the analyze agent then
   *hallucinated an entire fictional codebase* (`vars.rs`, `to_value`, a false "CRITICAL: consumer
   reimplements render") rather than reporting the tool failure. Earlier specs saw the same on research
   (fabricated crate versions) and critique/verify-tasks. **The audit subagents cannot be trusted on this
   project without main-thread re-verification.**
2. **The detect-changed-files.sh script is missing** (`.specify/scripts/bash/` lacks it), so both review
   skills' prescribed file-detection step errored (exit 127). Worked around with a direct merge-base diff,
   but it is a repeatable friction point.
3. **Provenance-lint design churned** because the original FR-018 was written against an imagined kernel
   "guard position" that spec 002 never built. A tighter 002→003 contract hand-off would have caught this
   at plan time, not analyze time.
