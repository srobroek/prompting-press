- `serde` and `serde_json` pinned to major `"1"` as plain version strings (per task instruction to write plain version strings; serde 1.x / serde_json 1.x are the current stable lines). `serde` carries `features = ["derive"]`.

## T002 — `rust-toolchain.toml`

```toml
# Explicit (non-floating) toolchain pin. Codegen determinism: cargo-typify +
# rustfmt output must be byte-stable across machines, so the rustfmt version is
# fixed by pinning the whole toolchain — a floating "stable" channel would drift
# rustfmt formatting and break the codegen-freshness CI gate (research D1/D4).
# MUST stay in lockstep with mise.toml (rust = "1.95.0") and Cargo.toml rust-version.

[toolchain]
channel = "1.95.0"
components = ["rustfmt", "clippy"]
profile = "minimal"
```

Notes / decisions:
- `channel = "1.95.0"` — explicit version, NOT the floating `"stable"` string. Matches `mise.toml` `rust = "1.95.0"` exactly. (Per repo note in `mise.toml`: `RUSTUP_TOOLCHAIN` set by mise overrides this file in rustup precedence, so both must remain identical.)
- `components = ["rustfmt", "clippy"]` — rustfmt required for deterministic Rust codegen (research D1/D4); clippy for linting.
- `profile = "minimal"` — keeps the toolchain lean (optional, included).
