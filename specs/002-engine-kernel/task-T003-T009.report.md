## Notes / findings

**Extra file found in blast radius**: `schemas/scripts/codegen-check.sh` (line 17, `RS=` variable)
hardcoded the consumer path and was not listed in D6's blast-radius map. It was updated as part
of T008 (same logical concern — the freshness gate). Without this fix `schemas:codegen-check`
produced exit 1 ("file missing on disk") even after the moon.yml dep and input path were corrected.

**generated `prompt_definition.rs` header update**: the committed file (moved from the consumer
crate) still carried `Regenerate with: crates/prompting-press/scripts/codegen.sh` in its static
header. The codegen script's HEADER constant was updated (T004), so re-running `:codegen` produced
a 1-line diff. The regenerated file was staged (`git add`) to align the index with the new canonical
output, making `schemas:codegen-check` green.

**Empty directory artifact**: `crates/prompting-press/src/generated/` remains as an empty filesystem
directory after the `git mv`. Git does not track empty directories; cargo is unaffected; no cleanup
needed.
