## ✅ What went well (keep doing)

- **The clarify→plan→refine discipline caught the FR-010 contradiction before code.** The
  "multi-variant must declare explicit default" requirement was structurally unreachable given the 001
  schema (`body` is always the default). Caught at plan time, ratified via refine, propagated. Zero
  rework in code.
- **Verify-at-spec-time on MiniJinja paid off twice.** A research subagent mis-claimed the latest
  version was 2.10.2; verifying directly against the crates.io sparse index + the 2.21.0 source tag
  confirmed 2.21 and validated the stable-API soundness (roadmap Q3). Later, the same source check
  proved `adjacent_loop_items` only gates `loop.previtem/nextitem` (critique E1) and that `debug` is a
  registered global even with the feature off (validating the env-derived allowlist, FR-020).
- **The review gate (review the diff, not the agent's summary) earned its keep repeatedly.** Caught:
  the `#[non_exhaustive]` over-reach on `KernelError`; the `looks_like_excluded_feature` heuristic that
  matched quoted keywords while MiniJinja emits bare `unknown statement <kw>` (misclassified all 6
  excluded features); the un-grounded critique/sync subagents (`tool_uses: 0`).
- **The agreement-check soundness held exactly as the constitution predicted** — `undeclared_variables(nested=false)`
  is stable-API, the include/macro exclusion (via disabled features) keeps it sound with zero unstable deps.
- **TDD per user story** (tests-first, red→green) produced 50 well-targeted tests; the SC↔test
  reconciliation (T035) confirmed every Success Criterion has a backing test.
- **Atomic phase commits** kept the highest-risk change (the generated-shape relocation) isolated and
  independently verified-green before kernel logic built on top.

## ⚠️ What could improve (start/stop)

- **The recurring subagent tool-channel glitch** (`tool_uses: 0`, empty reads) hit the fresh-context
  verify-tasks AND both sync agents — exactly the read-only audit roles where a fresh context is most
  valuable. Each time, the workaround was to run the audit on the main thread (losing the
  bias-isolation the fresh context was for). This is an environment issue, logged in the spec-001
  worklog too. **Worth a real fix or a documented fallback protocol.**
- **Two review subagents failed on wrong agent-type names** (`code-reviewer`, `debugger` — the real
  names are `comprehensive-review-code-reviewer`, `unit-testing-debugger`). Cost a relaunch. **A
  name-mapping note in the review orchestration would prevent it.**
- **The `constitution C-NN` misattribution regressed a fix spec-001 already made** (its analysis I2).
  The C-0N IDs are roadmap decisions, not constitution clauses. It crept back into the kernel doc-comments.
  **A lint or a CONTRIBUTING note ("cite roadmap decisions as 'roadmap decision C-NN', never
  'constitution C-NN'") would stop the recurrence.**
- **FR-029's fixture set was authored but not wired to a render-assertion** (verify F-01) — the
  fixtures existed and deserialized, but nothing rendered+asserted them. **A "regression fixtures must
  be consumed by an asserting test" check in the tasks template would catch this class.**
- **Backticks in `git commit -m "…"` double-quoted strings** got command-substituted twice
  (`debug` → empty). **Use single-quoted `-m` for messages containing backticks.**

## Improvement suggestions

- **IMP-001 [HIGH] Fix or document the subagent tool-channel glitch.** It defeated 3 fresh-context
  audits this cycle. Either fix the transport or codify the "run on main thread against objective
  evidence" fallback as the sanctioned protocol (it worked, but loses bias-isolation).
- **IMP-002 [MED] Add a `constitution C-NN` lint.** A trivial CI grep (`grep -rn "constitution C-0"`)
  over docs/comments would have caught CM-1 before review. The pattern is "roadmap decision C-NN".
- **IMP-003 [MED] Tasks template: regression fixtures need an asserting consumer.** FR-029's guard was
  unbacked because the fixtures + loader landed without a render-assert test. Note this in the template.
- **IMP-004 [LOW] Review-orchestration agent-name map.** Record the canonical agent-type names
  (`comprehensive-review-code-reviewer`, `unit-testing-debugger`, etc.) so review fan-out doesn't fail
  on a guessed name.
- **IMP-005 [LOW] Commit-message convention.** Single-quote `-m` when the body has backticks.
