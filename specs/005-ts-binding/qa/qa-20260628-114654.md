## Cross-language parity (Principle I — the structural guarantee)

The TS binding and the Python binding produce **byte-identical** `templateHash` + `renderHash` for the
same logical prompt + inputs (`greet`, `{name:"Ada", count:3}`):
`templateHash = 8de2d2a78db5c68e05d5d945b22fd4d6020e926372fa18936e635bcb583b570f`,
`renderHash = 9654e5f2a48422e7351c48d2776dc704e32051cc70e6926ed4b3d62b9311bc5c`. This is render parity
across two independent FFI bindings — the property spec 006's conformance corpus exists to lock down,
now demonstrated. (The Python call used the post-C-11 `data=` keyword form.)
