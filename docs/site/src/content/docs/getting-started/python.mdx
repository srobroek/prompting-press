## Write a prompt definition

A prompt definition is YAML, JSON, or TOML text — same shape, your choice of format. You read the text
however suits your app and hand it to the `from_*` factories; the library does no file I/O itself. The
`greet` definition used throughout this page:

<Tabs syncKey="def-format">
<TabItem label="YAML">
```yaml title="greet.yaml"
name: greet
role: user
body: "Hi {{ name }}, you have {{ count }} messages."
variables:
  name:
    type: string
    origin: trusted
  count:
    type: integer
    origin: trusted
```
</TabItem>
<TabItem label="JSON">
```json title="greet.json"
{
  "name": "greet",
  "role": "user",
  "body": "Hi {{ name }}, you have {{ count }} messages.",
  "variables": {
    "name": { "type": "string", "origin": "trusted" },
    "count": { "type": "integer", "origin": "trusted" }
  }
}
```
</TabItem>
<TabItem label="TOML">
```toml title="greet.toml"
name = "greet"
role = "user"
body = "Hi {{ name }}, you have {{ count }} messages."

[variables.name]
type = "string"
origin = "trusted"

[variables.count]
type = "integer"
origin = "trusted"
```
</TabItem>
</Tabs>

## Construct and render

### Step 1 — construct a `Prompt` (and validate)

Construct from text with a `from_*` factory, or from a shape object with `Prompt(...)` — either a
typed `PromptDefinition` or a plain `dict`. Every route runs the **same** validation immediately:

<Tabs syncKey="construct">
<TabItem label="From text">
```python
from prompting_press import Prompt

# You read the text however suits your app; the library does no file I/O itself.
with open("greet.yaml") as f:
    greet = Prompt.from_yaml(f.read())     # validates here, or raises
# from_json / from_toml parse the JSON / TOML forms into the same Prompt.

greet.name   # => "greet"
greet.body   # => "Hi {{ name }}, you have {{ count }} messages."
```
</TabItem>
<TabItem label="From a typed PromptDefinition">
```python
from prompting_press import Prompt, PromptDefinition

# PromptDefinition is the generated Pydantic shape — your editor/type-checker
# checks the fields, the role enum, and each variable's `origin` as you write it.
definition = PromptDefinition(
    name="greet",
    role="user",
    body="Hi {{ name }}, you have {{ count }} messages.",
    variables={
        "name":  {"type": "string",  "origin": "trusted"},
        "count": {"type": "integer", "origin": "trusted"},
    },
)

greet = Prompt(definition)   # same validation as the from_* factories
greet.name   # => "greet"
```
</TabItem>
<TabItem label="From a plain dict">
```python
from prompting_press import Prompt

# A plain dict works too — convenient when the shape comes from config you already
# parsed. The Rust loader validates the shape on the way in, the same as the typed form.
greet = Prompt({
    "name": "greet",
    "role": "user",
    "body": "Hi {{ name }}, you have {{ count }} messages.",
    "variables": {
        "name":  {"type": "string",  "origin": "trusted"},
        "count": {"type": "integer", "origin": "trusted"},
    },
})

greet.name   # => "greet"
```
</TabItem>
</Tabs>

Construction **validates immediately**: if the body had referenced `{{ usrname }}` — a variable
`variables` doesn't declare — construction would have raised `PromptRenderError` (code
`undefined_variable`), **before any render** (a malformed document raises `LoadError`). A
template/variable disagreement is caught at construction, never as a silent empty render.

### Step 2 — declare the typed Vars model

The values you render with are a Pydantic model you own. Its fields match the prompt's `variables`
(`name`, `count`), and its validators run before the kernel is touched:

```python
from pydantic import BaseModel, field_validator

class GreetVars(BaseModel):
    name: str
    count: int

    @field_validator("count")
    @classmethod
    def _non_negative(cls, v: int) -> int:
        if v < 0:
            raise ValueError("count must be non-negative")
        return v
```

### Step 3 — render, and read the result

Hand the `Prompt` from Step 1 the `GreetVars` from Step 2 — pass the **class** plus a `data` dict and
it validates + renders in one call:

```python
result = greet.render(GreetVars, data={"name": "Ada", "count": 3})

result.text           # => "Hi Ada, you have 3 messages."
result.variant        # => "default"  (same arm greet.body showed in Step 1)
result.template_hash  # => e.g. "9f2c…"  (64-char lowercase-hex SHA-256 of the template)
result.render_hash    # => e.g. "1ab4…"  (64-char lowercase-hex SHA-256 of result.text)
result.guard          # => None  (no guard requested)
```

`render(model=None, *, data=None, variant=None, guard=None)` — `data`/`variant`/`guard` are
keyword-only:
- Pass a Pydantic **class** + `data=` dict (as above), **or** a pre-built **instance** as `model`
  with no `data`.
- `variant=None` selects the default (root body) arm; `guard=None` opts out of the advisory guard.

### Complete example

```python
from prompting_press import Prompt
from pydantic import BaseModel, field_validator

class GreetVars(BaseModel):
    name: str
    count: int

    @field_validator("count")
    @classmethod
    def _non_negative(cls, v: int) -> int:
        if v < 0:
            raise ValueError("count must be non-negative")
        return v

# 1. Construct (validates here).
with open("greet.yaml") as f:
    greet = Prompt.from_yaml(f.read())

# 2 + 3. Render with the typed, Pydantic-validated vars.
result = greet.render(GreetVars, data={"name": "Ada", "count": 3})
print(result.text)           # Hi Ada, you have 3 messages.
print(result.template_hash)  # 64-char hex
```

## What `RenderResult` carries

| Field | Value |
|-------|-------|
| `text` | The rendered body text. |
| `name` | The prompt's name. |
| `variant` | The resolved variant name (`"default"` when none was selected). |
| `template_hash` | `SHA256(resolved variant template source)` — lowercase hex. |
| `render_hash` | `SHA256(rendered output text)` — lowercase hex. |
| `guard` | `str | None` — advisory guard text when opted in; `None` otherwise. |

## Error types

```python
from prompting_press import (
    PromptingPressError,
    PromptValidationError,
    PromptRenderError,
    LoadError,
)

try:
    result = greet.render(GreetVars, data={"name": "Ada", "count": -1})
except PromptValidationError as exc:
    for row in exc.errors:
        print(row.field, row.code, row.message)
        # "count"  "validation"  "count must be non-negative"
```

The exception hierarchy:

```
PromptingPressError          # base; carries .errors -> list[FieldError]
├── PromptValidationError    # Pydantic validation failed   (code "validation")
├── PromptRenderError        # kernel render failure         (codes: unknown_variant,
│                            #   undefined_variable, parse, render, excluded_feature)
├── UnknownPromptError       # a prompt name lookup failed  (code "unknown_prompt")
└── LoadError                # malformed YAML/JSON/TOML     (code "load")
```

Each `FieldError` carries `.field`, `.code`, and `.message`. Sensitive rejected values are
**never** echoed onto the error surface.
