## Write a prompt definition

A prompt definition is YAML, JSON, or TOML text — the same shape in any of the three formats. The
caller reads the text and hands it to the `from_*` factories; the library does no file I/O. The
`greet` definition used throughout this page:

<Tabs syncKey="def-format">
<TabItem label="YAML">
```yaml title="greet.yaml"
name: greet
role: user
body: "Hi {{ name }}, you have {{ count }} messages."
variables:
  name:
    type: string
    trusted: true
  count:
    type: integer
    trusted: true
```
</TabItem>
<TabItem label="JSON">
```json title="greet.json"
{
  "name": "greet",
  "role": "user",
  "body": "Hi {{ name }}, you have {{ count }} messages.",
  "variables": {
    "name": { "type": "string", "trusted": true },
    "count": { "type": "integer", "trusted": true }
  }
}
```
</TabItem>
<TabItem label="TOML">
```toml title="greet.toml"
name = "greet"
role = "user"
body = "Hi {{ name }}, you have {{ count }} messages."

[variables.name]
type = "string"
trusted = true

[variables.count]
type = "integer"
trusted = true
```
</TabItem>
</Tabs>

Construction **validates immediately**: if the body had referenced `{{ usrname }}` — a variable
`variables` doesn't declare — construction would have raised `PromptRenderError` (code
`undefined_variable`), **before any render** (a malformed document raises `LoadError`). A
template/variable disagreement is caught at construction, never as a silent empty render.

## What `RenderResult` carries

| Field | Value |
|-------|-------|
| `text` | The rendered body text. |
| `name` | The prompt's name. |
| `variant` | The resolved variant name (`"default"` when none was selected). |
| `template_hash` | `SHA256(resolved variant template source)` — lowercase hex. |
| `render_hash` | `SHA256(rendered output text)` — lowercase hex. |
| `guard` | `str | None` — advisory guard text when opted in; `None` otherwise. |

## Error types

<Code code={errorTypes} lang="python" title="getting-started_python_error_types.py" />

The exception hierarchy:

```
PromptingPressError          # base; carries .errors -> list[FieldError]
├── PromptValidationError    # Pydantic validation failed   (code "validation")
├── PromptRenderError        # kernel render failure         (codes: unknown_variant,
│                            #   undefined_variable, parse, render, excluded_feature)
└── LoadError                # malformed YAML/JSON/TOML     (code "load")
```

Each `FieldError` carries `.field`, `.code`, and `.message`. A rejected render value is **never**
echoed onto the error surface; a parse error carries template-syntax detail (no bound values).
