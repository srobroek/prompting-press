Package: `prompting-press` (PyPI). Import: `from prompting_press import ...`.
Python 3.12+. Single `abi3` wheel (`abi3-py312` тАФ one wheel covers CPython 3.12 and up).

The primary public type is `Prompt` тАФ an immutable, fully-validated prompt object. Validation is
owned by **Pydantic** at the binding boundary. Native Pydantic and Rust error types never leak
across FFI.

## `Prompt`

### Construction

All constructors validate fully тАФ they raise on any violation; never return `None` or a partial object.

#### `Prompt(shape, *, validators=None)`

```python
Prompt(shape: PromptDefinition | dict, *, validators: type[BaseModel] | None = None)
```

Primary validating constructor. `shape` is a `PromptDefinition` Pydantic model instance or a
plain `dict`. Both are reduced to JSON and routed through the Rust consumer.

`validators` is an optional Pydantic model **class**. When any `PromptVariable` in `shape.variables`
has `validation_required = True`, a covering `validators` class **must** be supplied тАФ construction
raises `PromptValidationError` naming the first uncovered variable if it is absent.

**Raises:**
- `PromptValidationError` тАФ shape/parse/agreement/reserved-name failure, or uncovered
  `validation_required` variable.
- `LoadError` тАФ `shape` could not be serialized to JSON.

#### `Prompt.from_yaml(text, *, validators=None)`

```python
@staticmethod
Prompt.from_yaml(text: str, *, validators: type[BaseModel] | None = None) -> Prompt
```

Parse already-read YAML text, then validate. The library reads no files.

**Raises:** `LoadError` for malformed YAML/shape; `PromptValidationError` for invariant violations.

#### `Prompt.from_json(text, *, validators=None)`

```python
@staticmethod
Prompt.from_json(text: str, *, validators: type[BaseModel] | None = None) -> Prompt
```

Same as `from_yaml` for JSON input.

#### `Prompt.from_toml(text, *, validators=None)`

```python
@staticmethod
Prompt.from_toml(text: str, *, validators: type[BaseModel] | None = None) -> Prompt
```

Same as `from_yaml` for TOML input. TOML parsing is done by the Rust consumer; no Python
`tomllib` dependency is used.

### Read-only properties

```python
prompt.name          # str
prompt.role          # str  тАФ "system" | "user" | "assistant"
prompt.body          # str  тАФ root body template source
prompt.variables     # dict тАФ { name: PromptVariable }
prompt.variants      # dict тАФ { name: PromptVariant }; empty when no named variants
prompt.output_model  # str | None
prompt.metadata      # dict
```

### `render`

```python
prompt.render(model=None, *, data=None, variant=None, guard=None) -> RenderResult
```

Validate-then-render. All keyword arguments after `model` are keyword-only.

Two call forms:
- **Class + data:** `render(GreetVars, data={"name": "Ada", "count": 3})` тАФ Pydantic
  `model.model_validate(data)` runs before any templating.
- **Instance:** `render(GreetVars(name="Ada", count=3))` тАФ instance is re-validated before
  templating.

`variant=None` selects the default (root body) arm. `guard=None` opts out of advisory guard text.

**Raises:**
- `PromptValidationError` тАФ Pydantic rejected the vars (before any templating).
- `PromptRenderError` тАФ kernel error (unknown variant, undefined variable, parse/render failure).

### `get_source`

```python
prompt.get_source(*, variant=None) -> str
```

Returns the unrendered template source for the given variant. Pure: no vars, no validation.
`variant=None` returns the root body source.

**Raises:** `PromptRenderError` тАФ unknown variant name.

### `check`

```python
prompt.check() -> CheckReport
```

Pure advisory lint. Returns a `CheckReport`. The only live finding is `untrusted_without_guard`.
Never renders, never mutates.

### `derive`

```python
prompt.derive(overlay: dict, *, validators=None) -> Prompt
```

Shallow-replace top-level fields from `overlay` (a dict), route the merged definition through
full re-validation, return a **new `Prompt`**. The original is untouched. `validators`
carries forward from `self` by default; pass `validators=NewModel` to override.

**Raises:** `PromptValidationError`, `PromptRenderError`, or `LoadError` тАФ same as construction.

## `CheckReport` and `Finding`

```python
report.passed()       # bool тАФ True iff no findings
report.is_empty()     # bool тАФ alias for passed()
bool(report)          # True iff there ARE findings (inverse of passed())
len(report)           # int тАФ number of findings
report.findings       # list[Finding]

# Finding fields:
finding.prompt        # str тАФ prompt name
finding.variant       # str | None
finding.kind          # str тАФ stable kind string (see table below)
finding.detail        # str тАФ human-readable, actionable description
```

`finding.kind` vocabulary:

| Kind | Meaning |
|------|---------|
| `"untrusted_without_guard"` | Untrusted/external variable with no guard key in `metadata`. The only kind `check()` returns. |

Hard invariants (undeclared variable, un-analyzable template, reserved variant name) are
enforced at construction and are never returned by `check()`.

## Exception hierarchy

```
PromptingPressError          # base; carries .errors: list[FieldError]
тФЬтФАтФА PromptValidationError    # Pydantic/construction validation failed  (code "validation")
тФЬтФАтФА PromptRenderError        # kernel render failure
тФВ                            #   codes: unknown_variant | undefined_variable | parse |
тФВ                            #          render | excluded_feature
тФФтФАтФА LoadError                # malformed YAML/JSON/TOML or shape error (code "load")
```

Each `FieldError`:
```python
error.field    # str тАФ offending field / path; "" when not field-specific
error.code     # str тАФ stable code (see table above)
error.message  # str тАФ human-readable message; render detail is scrubbed (no bound values);
               #        parse detail is preserved (template-syntax context, pre-binding)
```

## `Composition`

```python
from prompting_press import Composition, Message

comp = Composition()
comp.append(prompt, vars_instance)                        # validate eagerly
comp.append(prompt, vars_instance, variant="formal")      # with variant

# Or from a list of tuples:
comp = Composition.from_messages([
    (prompt1, vars1),
    (prompt2, vars2, "variant_name"),  # 3-tuple selects a variant
])

messages = comp.resolve()   # list[Message] in append order

# Message fields:
msg.role  # str тАФ from the Prompt definition's role
msg.text  # str тАФ rendered body text
```

`append` accepts a Pydantic model **instance**. Validation is eager тАФ a failure raises
`PromptValidationError` and nothing is stored.

## Content hashes vs. origin

| Concept | Where | What |
|---------|-------|------|
| `template_hash` / `render_hash` | `RenderResult` fields | SHA-256 content-addressed hashes of the template source and rendered output. Computed at render time. Byte-identical across all three language bindings. |
| `origin` | `variables[name].origin` | Per-variable trust tag (`"trusted"` / `"untrusted"` / `"external"`). Declarative metadata; not enforced by the kernel. |

These are not the same thing. See [Prompt definition shape](/prompting-press/reference/prompt-definition/) for `PromptVariable.origin` details.
